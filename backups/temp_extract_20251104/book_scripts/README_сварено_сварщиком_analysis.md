# Полный анализ заполнения таблицы `сварено_сварщиком`

## 📋 Общая информация

Таблица `сварено_сварщиком` создается и заполняется скриптом `scripts/database/create_svarenno_svarshchikom_table.py`. Это агрегированная таблица, которая группирует данные из таблицы `wl_china` по определенным критериям и вычисляет статистику по контролю качества сварных соединений.

## 🔄 Процесс создания и заполнения

### Этап 1: Предварительная обработка данных

**Метод:** `preprocess_welding_data()`
- **Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:113-137`

**Действия:**
1. Обнуление клейм сварщиков при пустой дате сварки:
   - Если `wl_china.Дата_сварки` пустая (NULL, '', или только пробелы), то:
     - `Клеймо_сварщика_корневой_слой` → NULL
     - `Клеймо_сварщика_заполнение_облицовка` → NULL

**SQL-запросы:**
```sql
UPDATE wl_china 
SET Клеймо_сварщика_корневой_слой = NULL
WHERE Дата_сварки IS NULL OR Дата_сварки = '' OR TRIM(Дата_сварки) = ''

UPDATE wl_china 
SET Клеймо_сварщика_заполнение_облицовка = NULL
WHERE Дата_сварки IS NULL OR Дата_сварки = '' OR TRIM(Дата_сварки) = ''
```

### Этап 3: Заполнение данными (Часть 1 - Сваренные швы)

**Метод:** `populate_table()` (первый INSERT)
- **Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:269-435`
- **Строки:** 275-373

#### Условия отбора данных из `wl_china`:

```sql
WHERE
    -- учитываем только сваренные строки (непустая дата)
    wc.Дата_сварки IS NOT NULL AND TRIM(wc.Дата_сварки) <> ''
    AND (
        wc.Клеймо_сварщика_корневой_слой IS NOT NULL AND wc.Клеймо_сварщика_корневой_слой <> ''
        OR wc.Клеймо_сварщика_заполнение_облицовка IS NOT NULL AND wc.Клеймо_сварщика_заполнение_облицовка <> ''
    )
    AND wc.Номер_сварного_шва IS NOT NULL
```

#### Группировка:

```sql
GROUP BY 
    wc.N_Линии,
    _Линия,  -- вычисляемое поле (очищенный N_Линии)
    wc.Номер_чертежа,
    клейма_сварщиков,  -- вычисляемое поле
    методы_сварки,    -- вычисляемое поле
    типы_сварных_швов -- вычисляемое поле
HAVING клейма_сварщиков IS NOT NULL
```

#### Заполнение полей:

1. **N_Линии**: Прямое копирование из `wc.N_Линии`

2. **_Линия**: Очищенная версия `N_Линии`
   ```sql
   CASE 
       WHEN wc.N_Линии IS NOT NULL THEN
           CASE 
               WHEN INSTR(CAST(wc.N_Линии AS TEXT), ' ') > 0 THEN 
                   SUBSTR(CAST(wc.N_Линии AS TEXT), 1, INSTR(CAST(wc.N_Линии AS TEXT), ' ') - 1)
               WHEN INSTR(CAST(wc.N_Линии AS TEXT), '(') > 0 THEN 
                   SUBSTR(CAST(wc.N_Линии AS TEXT), 1, INSTR(CAST(wc.N_Линии AS TEXT), '(') - 1)
               ELSE CAST(wc.N_Линии AS TEXT)
           END
       ELSE NULL
   END as _Линия
   ```
   **Логика:** Берется первая часть до пробела или скобки '('

3. **Номер_чертежа**: Прямое копирование из `wc.Номер_чертежа`

4. **Всего_сваренно_сварщиком**: `COUNT(wc.Номер_сварного_шва)`

5. **Не_Внесенно(не_сваренно)**: Устанавливается в `0` для сваренных швов

6. **клейма_сварщиков**: Объединение уникальных клейм через '/'
   ```sql
   (
       SELECT group_concat(part, '/')
       FROM (
           SELECT DISTINCT part
           FROM (
               SELECT trim(wc.Клеймо_сварщика_корневой_слой) as part 
               WHERE wc.Клеймо_сварщика_корневой_слой != ''
               UNION ALL
               SELECT trim(wc.Клеймо_сварщика_заполнение_облицовка) as part 
               WHERE wc.Клеймо_сварщика_заполнение_облицовка != ''
           )
           WHERE part IS NOT NULL AND part != ''
       )
   )
   ```
   **Источники:**
   - `Клеймо_сварщика_корневой_слой`
   - `Клеймо_сварщика_заполнение_облицовка`

7. **методы_сварки**: Объединение уникальных методов через '/'
   ```sql
   (
       SELECT group_concat(part, '/')
       FROM (
           SELECT DISTINCT part
           FROM (
               SELECT trim(wc.Метод_сварки_корневой_слой) as part 
               WHERE wc.Метод_сварки_корневой_слой IS NOT NULL AND wc.Метод_сварки_корневой_слой != ''
               UNION ALL
               SELECT trim(wc.Метод_сварки_заполнение_облицовка) as part 
               WHERE wc.Метод_сварки_заполнение_облицовка IS NOT NULL AND wc.Метод_сварки_заполнение_облицовка != ''
           )
           WHERE part IS NOT NULL AND part != ''
       )
   )
   ```
   **Источники:**
   - `Метод_сварки_корневой_слой`
   - `Метод_сварки_заполнение_облицовка`

8. **типы_сварных_швов**: Объединение уникальных типов через '/'
   ```sql
   (
       SELECT group_concat(part, '/')
       FROM (
           SELECT DISTINCT part
           FROM (
               SELECT trim(wc.Тип_соединения_российский_стандарт) as part 
               WHERE wc.Тип_соединения_российский_стандарт IS NOT NULL AND wc.Тип_соединения_российский_стандарт != ''
           )
           WHERE part IS NOT NULL AND part != ''
       )
   )
   ```
   **Источник:**
   - `Тип_соединения_российский_стандарт`

9. **исключено_по_типу**: Подсчет швов с типами У17 или У19
   ```sql
   SUM(CASE WHEN wc.Тип_соединения_российский_стандарт LIKE '%У17%' 
            OR wc.Тип_соединения_российский_стандарт LIKE '%У19%' 
       THEN 1 ELSE 0 END)
   ```

10. **Статистика ВИК** (из `Результаты_АКТ_ВИК`):
    ```sql
    заявлен_вик:     SUM(CASE WHEN wc.Результаты_АКТ_ВИК = 'Заказ отправлен' THEN 1 ELSE 0 END)
    годен_вик:       SUM(CASE WHEN wc.Результаты_АКТ_ВИК = 'годен' THEN 1 ELSE 0 END)
    не_годен_вик:    SUM(CASE WHEN wc.Результаты_АКТ_ВИК = 'Не годен' THEN 1 ELSE 0 END)
    не_подан_вик:    SUM(CASE WHEN wc.Результаты_АКТ_ВИК IS NULL OR wc.Результаты_АКТ_ВИК = '' 
                              OR wc.Результаты_АКТ_ВИК = 'None' THEN 1 ELSE 0 END)
    ```

11. **Статистика РК** (из `Результаты_Заключения_РК`):
    ```sql
    заявлен_рк:      SUM(CASE WHEN wc.Результаты_Заключения_РК = 'Заказ отправлен' THEN 1 ELSE 0 END)
    годен_рк:        SUM(CASE WHEN wc.Результаты_Заключения_РК = 'годен' THEN 1 ELSE 0 END)
    не_годен_рк:     SUM(CASE WHEN wc.Результаты_Заключения_РК = 'Не годен' THEN 1 ELSE 0 END)
    не_подан_рк:     0  -- рассчитывается позже
    ```

12. **Статистика PT** (из `Результаты_Заключения_PT`):
    ```sql
    заявлен_pt:      SUM(CASE WHEN wc.Результаты_Заключения_PT = 'Заказ отправлен' THEN 1 ELSE 0 END)
    годен_pt:        SUM(CASE WHEN wc.Результаты_Заключения_PT = 'годен' THEN 1 ELSE 0 END)
    не_годен_pt:     SUM(CASE WHEN wc.Результаты_Заключения_PT = 'Не годен' THEN 1 ELSE 0 END)
    не_подан_pt:     SUM(CASE WHEN wc.Результаты_Заключения_PT = 'Не годен' THEN 1 ELSE 0 END)
    ```
    ⚠️ **Примечание:** В коде есть потенциальная ошибка - `не_подан_pt` заполняется тем же условием, что и `не_годен_pt`.

13. **Статистика Стилоскопирование** (из `Результаты_Заключения_Стилоскопирование`):
    ```sql
    заявлен_стилоскопирование:  SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование = 'Заказ отправлен' THEN 1 ELSE 0 END)
    годен_стилоскопирование:    SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование = 'годен' THEN 1 ELSE 0 END)
    не_годен_стилоскопирование: SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование = 'Не годен' THEN 1 ELSE 0 END)
    не_подан_стилоскопирование: SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование IS NULL 
                                        OR wc.Результаты_Заключения_Стилоскопирование = '' 
                                        OR wc.Результаты_Заключения_Стилоскопирование = 'None' THEN 1 ELSE 0 END)
    ```

14. **Статистика МПД** (из `Результаты_Заключения_МПД`):
    ```sql
    заявлен_мпд:      SUM(CASE WHEN wc.Результаты_Заключения_МПД = 'Заказ отправлен' THEN 1 ELSE 0 END)
    годен_мпд:        SUM(CASE WHEN wc.Результаты_Заключения_МПД = 'годен' THEN 1 ELSE 0 END)
    не_годен_мпд:     SUM(CASE WHEN wc.Результаты_Заключения_МПД = 'Не годен' THEN 1 ELSE 0 END)
    не_подан_мпд:     SUM(CASE WHEN wc.Результаты_Заключения_МПД IS NULL 
                            OR wc.Результаты_Заключения_МПД = '' 
                            OR wc.Результаты_Заключения_МПД = 'None' THEN 1 ELSE 0 END)
    ```

### Этап 5: Расчет плановых значений контроля

После вставки данных выполняются UPDATE-запросы для расчета плановых значений контроля качества.

#### 5.1. Расчет РК (Радиографический контроль)

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:436-451`

**Формула:**
```sql
РК = MAX(
    CAST(((Всего_сваренно_сварщиком - не_годен_рк - исключено_по_типу) * RT% / 100 + 99) AS INT) - годен_рк,
    0
)
```

**Источник процента:** Таблица `основнаяНК`, поле `"РК (Радиографический контроль) / RT"` по ключу `_Линия = "Номер линии / Line No"`

**Логика:**
1. База: `Всего_сваренно_сварщиком - не_годен_рк - исключено_по_типу`
2. Вычисляется `ceil(база * процент / 100)` через формулу `(база * процент + 99) / 100`
3. Вычитается `годен_рк`
4. Результат не может быть меньше 0

#### 5.2. Пересчет не_подан_рк

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:453-460`

**Формула:**
```sql
не_подан_рк = MAX(РК - заявлен_рк, 0)
```

**Логика:** Разница между планом РК и заявленными, не меньше 0.

#### 5.3. Расчет Стило(PMI)

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:462-516`

**Формула (сложная):**

**Если в основнойНК для линии указано '*' в поле PMI:**
```sql
Стило(PMI) = 2
```

**Иначе:**
```sql
Стило(PMI) = MAX(
    ceil((Всего_сваренно_сварщиком - не_годен_рк) * процент_PMI / 100) - годен_стилоскопирование,
    0
)
```

**Источник процента:** Таблица `основнаяНК`, поле `"СТ (Стилоскоп) Стилоскопирование / PMI"` по ключу `_Линия = "Номер линии / Line No"`

**Логика:**
1. Проверяется наличие '*' - если есть, значение = 2
2. Иначе вычисляется процент, очищается от '%' и ',' → '.'
3. База: `Всего_сваренно_сварщиком - не_годен_рк`
4. Применяется формула ceil через `(база * процент + 99) / 100`
5. Вычитается `годен_стилоскопирование`
6. Результат не меньше 0

#### 5.4. Пересчет не_подан_стилоскопирование

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:518-525`

**Формула:**
```sql
не_подан_стилоскопирование = MAX("Стило(PMI)" - заявлен_стилоскопирование, 0)
```

#### 5.5. Расчет ПВК (PT)

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:527-542`

**Формула:**
```sql
ПВК = MAX(
    CAST(((Всего_сваренно_сварщиком - не_годен_рк) * PT% / 100 + 99) AS INT) - годен_pt,
    0
)
```

**Источник процента:** Таблица `основнаяНК`, поле `"ПВК (Контроль проникающими веществами) / PT"` по ключу `_Линия = "Номер линии / Line No"`

**Логика:**
1. База: `Всего_сваренно_сварщиком - не_годен_рк`
2. Вычисляется `ceil(база * процент / 100)`
3. Вычитается `годен_pt`
4. Результат не меньше 0

#### 5.6. Пересчет не_подан_pt

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:544-551`

**Формула:**

#### 5.7. Расчет МПД (MT)

**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py:553-568`

**Формула:**

**Источник процента:** Таблица `основнаяНК`, поле `"МПК (Магнитопорошковый контроль) / MT"` по ключу `_Линия = "Номер линии / Line No"`

**Логика:**

### Этап 6: Заполнение справочных полей

```sql
SET 
    "No TEST-PACK" = COALESCE(
        (SELECT lt."No TEST-PACK" FROM LstTp_12460 lt WHERE lt.Линия = ss.N_Линии LIMIT 1),

    "№ ИЧ" = COALESCE(
        (SELECT lt."№ ИЧ" FROM LstTp_12460 lt WHERE lt.Линия = ss.N_Линии LIMIT 1),
        (SELECT lt."№ ИЧ" FROM LstTp_12460 lt WHERE [очищенная_Линия] = ss._Линия LIMIT 1)
```

## ⚠️ Важные замечания и потенциальные проблемы

### 1. Ошибка в заполнении не_подан_pt

**Строка 344:** `не_подан_pt` заполняется тем же условием, что и `не_годен_pt`:
```sql
SUM(CASE WHEN wc.Результаты_Заключения_PT = 'Не годен' THEN 1 ELSE 0 END) as не_подан_pt,
```

**Должно быть:**
```sql
SUM(CASE WHEN wc.Результаты_Заключения_PT IS NULL OR wc.Результаты_Заключения_PT = '' 
         OR wc.Результаты_Заключения_PT = 'None' THEN 1 ELSE 0 END) as не_подан_pt,
```

**Однако:** Это поле пересчитывается позже (строка 547-549), так что фактическое значение корректное:
```sql
UPDATE сварено_сварщиком
SET не_подан_pt = MAX(ПВК - COALESCE(заявлен_pt, 0), 0)
```

### 2. Особенность расчета не_подан_рк

Изначально `не_подан_рк` устанавливается в 0 при INSERT (строка 340), затем пересчитывается из формулы `РК - заявлен_рк` (строка 456-459).

### 3. Очистка _Линия

Поле `_Линия` получается из `N_Линии` путем удаления части после пробела или скобки. Это может привести к объединению разных линий в одну группу.

**Пример:**
- `N_Линии = "12 (магистраль)"` → `_Линия = "12"`
- `N_Линии = "12 (ответвление)"` → `_Линия = "12"`
- Обе записи попадут в одну группу.

### 4. Группировка по клеймам, методам и типам

Таблица группирует данные не только по линии и чертежу, но и по комбинации клейм, методов и типов швов. Это означает, что для одной линии и чертежа может быть несколько записей, если различаются:
- Клейма сварщиков
- Методы сварки
- Типы сварных швов

### 5. Зависимость от основнойНК

Расчет плановых значений зависит от наличия записей в таблице `основнаяНК`. Если для линии нет записи, проценты будут равны 0, что приведет к нулевым плановым значениям.
