# Анализ проблемы с расчетом "Стило(PMI)" и плановым остаточным значением

## 🔍 Обнаруженные проблемы

### Проблема 1: Несоответствие формулы ceil

**Место:** `scripts/database/create_svarenno_svarshchikom_table.py`, строки 477-509

**Проблема:**

Для расчета `"Стило(PMI)"` используется **сложная формула ceil** через проверку дробной части:

```sql
CASE 
    WHEN (база * процент / 100.0) > CAST((база * процент / 100.0) AS INT)
    THEN CAST((база * процент / 100.0) AS INT) + 1
    ELSE CAST((база * процент / 100.0) AS INT)
END
```

**Но для других полей (РК, ПВК, МПД)** используется **стандартная формула ceil**:
```sql
CAST(((база * процент) + 99) / 100 AS INT)
```

**Последствие:** Разные формулы ceil могут давать разные результаты для некоторых значений!

**Пример несоответствия:**
- База = 95, процент = 15
- Стандартная формула: `(95 * 15 + 99) / 100 = 1524 / 100 = 15` ✅
- Сложная формула: `95 * 15 / 100.0 = 14.25`, проверка: `14.25 > 14` → `14 + 1 = 15` ✅
- В этом случае совпадает, но могут быть ошибки округления!

**Рекомендация:** Унифицировать формулу ceil для всех полей.

### Проблема 2: Формула не использует стандартную ceil с (+99)

**Строки 479-509:** Формула ceil реализована через CASE WHEN, а не через `(value + 99) / 100`

**Сравнение с другими полями:**

**РК (строка 441):**
```sql
CAST(((база * процент) + 99) / 100 AS INT)
```

**ПВК (строка 532):**
```sql
CAST(((база * процент) + 99) / 100 AS INT)
```

**МПД (строка 558):**
```sql
CAST(((база * процент) + 99) / 100 AS INT)
```

**Стило(PMI) (строки 479-509):**
```sql
CASE 
    WHEN ((база * процент) / 100.0) > CAST(((база * процент) / 100.0) AS INT)
    THEN CAST(((база * процент) / 100.0) AS INT) + 1
    ELSE CAST(((база * процент) / 100.0) AS INT)
END
```

**Проблема:** Разная реализация ceil может привести к расхождениям!

### Проблема 3: Возможные варианты значений "годен" не учитываются

**Место:** `scripts/database/create_svarenno_svarshchikom_table.py`, строка 346

**Текущая формула:**
```sql
SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование = 'годен' THEN 1 ELSE 0 END)
```

**Проблема:** Проверяется только точное совпадение `= 'годен'` (строчными буквами).

**Возможные варианты значений:**
- `'годен'` ✅ (учитывается)
- `'Годен'` ❌ (не учитывается - с заглавной буквы)
- `'ГОДЕН'` ❌ (не учитывается - все заглавные)
- `'годен '` ❌ (не учитывается - с пробелом)
- `' годен'` ❌ (не учитывается - с пробелом в начале)

**Рекомендация:** Использовать TRIM и UPPER для нормализации:
```sql
SUM(CASE WHEN UPPER(TRIM(COALESCE(wc.Результаты_Заключения_Стилоскопирование, ''))) = 'ГОДЕН' 
    THEN 1 ELSE 0 END)
```

### Проблема 4: Комментарий не соответствует коду для `не_подан_стилоскопирование`

**Место:** `scripts/database/create_svarenno_svarshchikom_table.py`, строки 518-522

**Комментарий (строка 518):**
```python
# Пересчет не_подан_стилоскопирование = Стило(PMI) - заявлен_стилоскопирование - годен_стилоскопирование
```

**Реальный код (строка 522):**
```sql
SET не_подан_стилоскопирование = MAX("Стило(PMI)" - COALESCE(заявлен_стилоскопирование, 0), 0)
```

**Проблема:** В коде **НЕ вычитается** `годен_стилоскопирование`, хотя в комментарии это указано!

**Сравнение с другими полями:**

**не_подан_рк (строка 457):**
```sql
SET не_подан_рк = MAX(РК - COALESCE(заявлен_рк, 0), 0)
```
(также не вычитается годен_рк)

**не_подан_pt (строка 548):**
```sql
SET не_подан_pt = MAX(ПВК - COALESCE(заявлен_pt, 0), 0)
```
(также не вычитается годен_pt)

**не_подан_мпд (строка 574):**
```sql
SET не_подан_мпд = MAX(МПД - COALESCE(заявлен_мпд, 0), 0)
```
(также не вычитается годен_мпд)

**Вывод:** Логика согласована - `не_подан` считается как разница между планом и заявленными, а не между планом и годными. Но комментарий вводит в заблуждение.

**Однако:** Возможно, правильная логика должна быть:
- `не_подан = план - заявлен` (текущая реализация)
- ИЛИ `не_подан = план - годен` (как указано в комментарии)

## 🔧 Рекомендации по исправлению

### Исправление 1: Унифицировать формулу ceil для Стило(PMI)

**Было (строки 477-509):**
```sql
CASE 
    WHEN ((база * процент) / 100.0) > CAST((...) AS INT)
    THEN CAST((...) AS INT) + 1
    ELSE CAST((...) AS INT)
END
```

**Должно быть (как для РК/ПВК/МПД):**
```sql
CAST(((MAX(ss.Всего_сваренно_сварщиком - ss.не_годен_рк, 0)) * COALESCE((...), 0) + 99) / 100 AS INT)
```

**Пример исправления:**
```sql
UPDATE сварено_сварщиком AS ss
SET "Стило(PMI)" = MAX(
    CASE 
        WHEN EXISTS (
            SELECT 1 FROM основнаяНК он
            WHERE он."Номер линии / Line No" = ss._Линия
              AND он."СТ (Стилоскоп) Стилоскопирование / PMI" LIKE '%*%'
        ) THEN 2
        ELSE CAST((
            (MAX(ss.Всего_сваренно_сварщиком - ss.не_годен_рк, 0)) * COALESCE((
                SELECT CAST(REPLACE(REPLACE(он."СТ (Стилоскоп) Стилоскопирование / PMI", '%', ''), ',', '.') AS REAL)
                FROM основнаяНК он
                WHERE он."Номер линии / Line No" = ss._Линия
                LIMIT 1
            ), 0) + 99) / 100 AS INT
        )
    END - COALESCE(ss.годен_стилоскопирование, 0),
    0
)
```
