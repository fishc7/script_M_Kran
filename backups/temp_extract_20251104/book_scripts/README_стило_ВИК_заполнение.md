# Детальное описание заполнения столбцов: "Стило(PMI)", годен_стилоскопирование, ВИК

## 📋 Общая информация

Эти три столбца заполняются на разных этапах создания таблицы `сварено_сварщиком`. Рассмотрим каждый подробно.

---

## 1️⃣ Столбец `годен_стилоскопирование`

### Когда заполняется:
**Этап:** При первом INSERT (заполнение сваренных швов)
**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py`, строка 346

### Логика заполнения:

```sql
SUM(CASE WHEN wc.Результаты_Заключения_Стилоскопирование = 'годен' THEN 1 ELSE 0 END) as годен_стилоскопирование
```

### Источник данных:
- **Таблица:** `wl_china`
- **Поле:** `Результаты_Заключения_Стилоскопирование`

### Как работает:

1. Для каждой группы (N_Линии, _Линия, Номер_чертежа, клейма, методы, типы)
2. Подсчитываются все записи из `wl_china`, где:
   - `Результаты_Заключения_Стилоскопирование = 'годен'` (точное совпадение)
   - И запись входит в данную группу (GROUP BY)

3. Результат: **количество швов, которые прошли стилоскопирование с результатом "годен"**

### Пример:

**Данные в wl_china:**
| Номер_сварного_шва | Результаты_Заключения_Стилоскопирование |
|---------------------|------------------------------------------|
| ШВ-001              | годен                                    |
| ШВ-002              | годен                                    |
| ШВ-003              | Не годен                                 |
| ШВ-004              | Заказ отправлен                          |
| ШВ-005              | (NULL)                                   |

**Результат в сварено_сварщиком:**
```
годен_стилоскопирование = 2  (ШВ-001 и ШВ-002)
```

### ⚠️ Важно:
- Значение заполняется **один раз** при INSERT
- **Не пересчитывается** в дальнейшем
- Это фактическое количество швов, которые уже прошли стилоскопирование успешно

## 2️⃣ Столбец `"Стило(PMI)"` (плановое значение)

### Когда заполняется:
**Этап:** UPDATE после INSERT (расчет плановых значений)
**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py`, строки 462-516

### Логика заполнения:

**Формула в упрощенном виде:**
```sql
"Стило(PMI)" = MAX(
    IF (в основнойНК для линии указано '*') THEN 2
    ELSE ceil((Всего_сваренно_сварщиком - не_годен_рк) * процент_PMI / 100)
    END
    - годен_стилоскопирование,
    0
)
```

### Источники данных:

1. **Из таблицы `сварено_сварщиком`** (уже заполненные):
   - `Всего_сваренно_сварщиком`
   - `не_годен_рк`
   - `годен_стилоскопирование` (заполнено ранее)

2. **Из таблицы `основнаяНК`**:
   - Поле: `"СТ (Стилоскоп) Стилоскопирование / PMI"`
   - Ключ сопоставления: `основнаяНК."Номер линии / Line No" = сварено_сварщиком._Линия`

### Детальная логика:

#### Вариант 1: Если в `основнаяНК` указано '*' 

```sql
CASE 
    WHEN EXISTS (
        SELECT 1 FROM основнаяНК он
        WHERE он."Номер линии / Line No" = ss._Линия
          AND он."СТ (Стилоскоп) Стилоскопирование / PMI" LIKE '%*%'
    ) THEN 2
```

**Результат:** `"Стило(PMI)" = MAX(2 - годен_стилоскопирование, 0)`

**Пример:**
- `годен_стилоскопирование = 1`
- Результат: `MAX(2 - 1, 0) = 1`

#### Вариант 2: Если указан процент (например, "10%", "15,5%")

```sql
ELSE (
    -- ceil(база * процент / 100) - годен_стилоскопирование
    -- где база = (Всего_сваренно_сварщиком - не_годен_рк)
)
```

**Шаги расчета:**

1. **Извлечение процента:**
   ```sql
   SELECT CAST(REPLACE(REPLACE(он."СТ (Стилоскоп) Стилоскопирование / PMI", '%', ''), ',', '.') AS REAL)
   FROM основнаяНК он
   WHERE он."Номер линии / Line No" = ss._Линия
   ```
   - Удаляется символ '%'
   - Запятая заменяется на точку (для десятичных процентов)
   - Приводится к REAL (число)

2. **Вычисление базы:**
   ```sql
   база = MAX(Всего_сваренно_сварщиком - не_годен_рк, 0)
   ```
   - Из общего количества сваренных швов вычитаются негодные по РК
   - Результат не может быть отрицательным

3. **Вычисление ceil (округление вверх):**
   ```sql
   ceil(база * процент / 100)
   ```
   - Формула реализации ceil в SQLite:
   ```sql
   CASE 
       WHEN (база * процент / 100.0) > CAST((база * процент / 100.0) AS INT)
       THEN CAST((база * процент / 100.0) AS INT) + 1
       ELSE CAST((база * процент / 100.0) AS INT)
   END
   ```

4. **Вычитание уже годных:**
   ```sql
   результат = ceil(...) - годен_стилоскопирование
   ```

5. **Ограничение снизу:**
   ```sql
   "Стило(PMI)" = MAX(результат, 0)
   ```

### Полный пример расчета (Вариант 2):

**Исходные данные:**
- `Всего_сваренно_сварщиком = 100`
- `не_годен_рк = 5`
- `годен_стилоскопирование = 10`
- В `основнаяНК` для линии указано: `"СТ (Стилоскоп) Стилоскопирование / PMI" = "15%"`

**Расчет:**
1. База: `MAX(100 - 5, 0) = 95`
2. Процент: `15` (из "15%")
3. План без вычета: `ceil(95 * 15 / 100) = ceil(14.25) = 15`
4. План с учетом годных: `15 - 10 = 5`
5. Финальный результат: `MAX(5, 0) = 5`

**Итог:** `"Стило(PMI)" = 5`

### Вариант 3: Нет данных в `основнаяНК`

Если для линии нет записи в `основнаяНК`:
- Процент = 0 (из COALESCE)
- База * 0 / 100 = 0
- `"Стило(PMI)" = MAX(0 - годен_стилоскопирование, 0) = 0`

### ⚠️ Важно:
- `"Стило(PMI)"` - это **плановое остаточное значение**, сколько еще нужно проверить
- Учитывает уже проверенные (`годен_стилоскопирование`)
- Не может быть отрицательным

## 3️⃣ Столбец `ВИК`

### Когда заполняется:
**Этап:** UPDATE после INSERT
**Файл:** `scripts/database/create_svarenno_svarshchikom_table.py`, строка 581

### Логика заполнения:

**SQL-запрос:**
```sql
UPDATE сварено_сварщиком SET ВИК = COALESCE(заявлен_вик, 0)
```

### Источник данных:

**Из таблицы `сварено_сварщиком`** (уже заполненное при INSERT):
- `заявлен_вик` - заполнено на этапе INSERT из `wl_china`

### Как работает:

1. `заявлен_вик` было заполнено ранее при INSERT:
   ```sql
   SUM(CASE WHEN wc.Результаты_АКТ_ВИК = 'Заказ отправлен' THEN 1 ELSE 0 END) as заявлен_вик
   ```
   
2. `ВИК` просто копирует значение `заявлен_вик`, заменяя NULL на 0:
   ```sql
   ВИК = COALESCE(заявлен_вик, 0)
   ```

### Пример:

**При INSERT:**
- `заявлен_вик = 15` (15 швов со статусом "Заказ отправлен")

**После UPDATE:**
- `ВИК = 15`

### ⚠️ Важно:
- `ВИК` **равен** количеству заявленных швов на визуальный контроль
- Это просто копия `заявлен_вик` для удобства
- В отличие от `"Стило(PMI)"`, **не является плановым остаточным значением**, а показывает сколько заявлено

## 🔍 Детальные примеры

### Пример 1: Полный расчет для одной группы

**Исходные данные в `wl_china` (группированные):**
- Линия: "12"
- Номер_чертежа: "ISO-001"
- Всего швов в группе: 50
- `Результаты_Заключения_Стилоскопирование`:
  - "годен": 10 швов
  - "Не годен": 2 шва
  - "Заказ отправлен": 5 швов
  - NULL: 33 шва

**Данные в `основнаяНК`:**
- `"Номер линии / Line No" = "12"`
- `"СТ (Стилоскоп) Стилоскопирование / PMI" = "20%"`

**Данные для расчета:**
- `Всего_сваренно_сварщиком = 50`
- `не_годен_рк = 3` (допустим)

**Расчет:**

#### 1. `годен_стилоскопирование` (при INSERT):
```
= COUNT(швы где Результаты = 'годен')
= 10
```

#### 2. `"Стило(PMI)"` (при UPDATE):
1. Проверка '*': Нет (в основнойНК указано "20%")
2. База: `MAX(50 - 3, 0) = 47`
3. Процент: `20`
4. План: `ceil(47 * 20 / 100) = ceil(9.4) = 10`
5. С учетом годных: `10 - 10 = 0`
6. Результат: `MAX(0, 0) = 0`

**Итог:** `"Стило(PMI)" = 0` (план выполнен)

#### 3. `заявлен_вик` (при INSERT):
```sql
SUM(CASE WHEN wc.Результаты_АКТ_ВИК = 'Заказ отправлен' THEN 1 ELSE 0 END)
```
Допустим = 15

#### 4. `ВИК` (при UPDATE):
```
= COALESCE(заявлен_вик, 0)
= 15
```

**Финальные значения:**
- `годен_стилоскопирование = 10`
- `"Стило(PMI)" = 0`
- `ВИК = 15`

### Пример 2: Символ '*' в основнойНК

**Данные в `основнаяНК`:**
- `"СТ (Стилоскоп) Стилоскопирование / PMI" = "*"` (или содержит '*')

**Расчет `"Стило(PMI)"`:**
```
IF (найден '*') THEN
    план = 2
ELSE
    план = расчет по проценту
END

"Стило(PMI)" = MAX(план - годен_стилоскопирование, 0)
```

**Пример:**
- `годен_стилоскопирование = 1`
- `"Стило(PMI)" = MAX(2 - 1, 0) = 1`

**Смысл:** Если указан '*', планируется проверка минимум 2 швов.

## 🎯 Резюме

### `годен_стилоскопирование`:
- **Что:** Количество швов, которые **уже прошли** стилоскопирование успешно
- **Когда:** Заполняется при INSERT
- **Источник:** Фактические данные из `wl_china`

### `"Стило(PMI)"`:
- **Что:** Плановое **остаточное** количество швов, которые еще нужно проверить стилоскопированием
- **Когда:** Рассчитывается при UPDATE
- **Источник:** 
  - Процент из `основнаяНК`
  - Или специальное значение 2, если указан '*'
  - Учитывает уже проверенные (`годен_стилоскопирование`)

### `ВИК`:
- **Что:** Количество швов, **заявленных** на визуальный контроль
- **Когда:** Копируется при UPDATE
- **Источник:** Поле `заявлен_вик`, заполненное из `wl_china`
