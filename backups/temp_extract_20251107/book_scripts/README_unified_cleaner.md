# Универсальный скрипт очистки данных

## Описание
`unified_data_cleaner.py` - это универсальный скрипт для полной очистки данных в таблице `wl_report_smr` базы данных SQLite.

## Функциональность

### Этап 1: Очистка столбца КЛЕЙМО
- **Удаление скобок и запятых**: Удаляет символы `(`, `)`, `,` из значений
- **Замена длинных текстовых значений**: Заменяет значения содержащие паттерны (`стык заварен`, `есть в ЖСР`, `В ЖСР С`, `дубль`, `перевар`, `OEES`) на соответствующие значения из столбца `Факт`
- **Замена пустых значений**: Заполняет пустые значения в `КЛЕЙМО` данными из столбца `Факт`
- **Стандартизация формата**: 
  - Удаляет все пробелы
  - Преобразует в верхний регистр
  - Заменяет буквы `O`/`о` на цифру `0`

### Этап 2: Очистка столбца Факт
- **Удаление значений с "Перевар"**: Удаляет все значения содержащие слово "Перевар" (без учета регистра и пробелов)

### Этап 3: Очистка множественных столбцов
Очищает столбцы `КЛЕЙМО_1`, `Ревизия`, `Unnamed:_16` от нежелательных данных по списку:
- `ткс`, `TKC`, `ПЕРЕВАР`
- Даты в различных форматах
- Специальные значения: `Долг`, `Врезка`, `Ремонт`, `45772`
- Описательные фразы

### Этап 4: Создание столбца Фактическое_Клеймо
Создает новый столбец `Фактическое_Клеймо` с данными по приоритету:
- **Приоритет 1**: `Unnamed:_16` - если не пустой
- **Приоритет 2**: `Ревизия` - если `Unnamed:_16` пустой
- **Приоритет 3**: `КЛЕЙМО_1` - если предыдущие пустые
- **Приоритет 4**: `КЛЕЙМО` - если все предыдущие пустые

## Использование

```bash
cd scripts
python unified_data_cleaner.py
```

Или из корневой директории проекта:
```bash
python scripts/unified_data_cleaner.py
```

## Требования
- Python 3.6+
- pandas
- sqlite3 (встроенный модуль)

## Результаты
Скрипт создает:
1. **Лог-файл**: `unified_cleaning_YYYYMMDD_HHMMSS.log` - подробный лог выполнения (опционально)

## Безопасность
- Все изменения выполняются в транзакции
- При ошибке выполняется откат (rollback)
- Создается резервная копия перед изменениями (рекомендуется)

## Статистика
После выполнения выводится статистика по каждому столбцу:
- Количество записей с данными
- Количество пустых записей
- Процент заполненности
