### 2.3 Внедрение паттерна Repository

**Проблема**:  
Бизнес-логика смешана с логикой доступа к данным.

**Решение**:
Создать слой Repository для инкапсуляции логики работы с данными:

```python
# scripts/core/repositories/weld_repository.py
class WeldRepository:
    def __init__(self, dao: WeldDAO):
        self.dao = dao
    
    def find_by_iso(self, iso: str):
        return self.dao.find_by_iso(iso)
    
    def find_duplicates(self):
        # Бизнес-логика поиска дубликатов
        return self.dao.find_duplicates()
```

## 4. Оптимизация производительности

### 4.1 Оптимизация работы с БД

**Проблема**:  
Многие скрипты выполняют запросы в циклах, что неэффективно.

**Решение**:
1. Использовать batch-операции (INSERT/UPDATE множества записей)
2. Использовать транзакции для группировки операций
3. Добавить индексы на часто используемые поля
4. Использовать prepared statements

**Пример**:
```python
# Плохо
for row in data:
    cursor.execute("INSERT INTO table VALUES (?)", (row,))

# Хорошо
cursor.executemany("INSERT INTO table VALUES (?)", data)
conn.commit()
```

**Приоритет**: 🟠 **ВЫСОКИЙ**

**Оценка времени**: 8-12 часов
