# 1. Обзор проекта

## 🎯 Назначение

**M_Kran** - это комплексная система управления данными сварочных работ для промышленных проектов. Система предназначена для:

- **Обработки данных** сварочных работ из различных источников
- **Анализа и контроля качества** сварочных соединений
- **Управления неразрушающим контролем (НК)** - ВИК, РК, ПВК, МПД, Стилоскопирование
- **Отслеживания состояния** сварочных работ в реальном времени
- **Генерации отчетов** и аналитики
- **Интеграции данных** из различных систем (Excel, базы данных, веб-отчеты)

## 📊 Основные характеристики

| Параметр | Значение |
|----------|----------|
| **Тип системы** | Веб-приложение с модульной архитектурой |
| **База данных** | SQLite с 32+ таблицами |
| **Размер проекта** | ~383 МБ |
| **Количество записей** | 100,000+ записей сварочных работ |
| **Языки программирования** | Python 3.8+, JavaScript, HTML/CSS |
| **Фреймворк** | Flask 2.3.3 |
| **Архитектура** | Модульная, масштабируемая |
| **Количество модулей** | 8 основных модулей |
| **Количество скриптов** | 100+ скриптов обработки данных |

## 🎯 Ключевые возможности

### 📥 Импорт данных
- Загрузка данных из Excel файлов
- Импорт из различных источников (НГС, СМР, Китайские подрядчики)
- Автоматическая обработка и валидация данных

### 🔄 Обработка данных
- Очистка и нормализация данных
- Поиск и обработка дубликатов
- Синхронизация между таблицами
- Трансформация данных

### 📊 Аналитика и отчетность
- Статистика по сварочным работам
- Анализ качества сварки
- Отслеживание прогресса работ
- Генерация отчетов в Excel и HTML

### 🔍 Контроль качества
- Управление неразрушающим контролем (НК)
- Отслеживание результатов ВИК, РК, ПВК, МПД, Стилоскопирования

## 🔧 Модульная архитектура

### Модуль конфигурации (`modules/config.py`)

Централизованная конфигурация системы:
- Настройки Flask приложения
- Пути к файлам и папкам
- Настройки кодировки
- Конфигурация базы данных
- Настройки безопасности

### Модуль базы данных (`modules/database/db_manager.py`)

Класс `DatabaseManager` для работы с БД:
- Подключение к базе данных
- Выполнение запросов
- Получение статистики
- Работа с таблицами

### Модуль маршрутов (`modules/routes/`)

4 модуля маршрутов:
1. **main_routes.py** - Главные страницы (главная, база данных, таблицы)
2. **files_routes.py** - Работа с файлами (загрузка, экспорт, удаление)
3. **scripts_routes.py** - Управление скриптами (запуск, статус)
4. **system_routes.py** - Системные функции (логи, бэкапы)

### Модуль API (`modules/api/`)

2 модуля API:
1. **api_routes.py** - Основные API эндпоинты
2. **extended_api_routes.py** - Расширенные API функции

### Модуль утилит (`modules/utils/`)

Утилиты для работы с данными:
- Форматирование чисел
- Обработка текста
- Валидация данных

# 6. ETL процессы и скрипты

## 🔄 ETL архитектура

Система использует ETL (Extract, Transform, Load) подход для обработки данных:

```
EXTRACT (Извлечение)
    ↓
TRANSFORM (Преобразование)
    ↓
LOAD (Загрузка)
```

## 📥 Extract - Извлечение данных

### Ежедневные скрипты (приоритет 1-10)

| Скрипт | Приоритет | Описание | Источник | Целевая таблица |
|--------|-----------|----------|----------|-----------------|
| `load_lnk_data.py` | 1 | Журнал НК НГС | `Журнал ЛНК_*.xlsx` | `logs_lnk` |
| `load_staff_titles_M_Kran.py` | 2 | Расстановка персонала | `Сварка *.xlsx` | `Staff_Titles_M_Kran` |
| `load_ndt_findings_transmission_register.py` | 3 | Реестр заключений НГС | Excel файлы | `NDT_Findings_Transmission_Register` |
| `load_wl_report_smr_web.py` | 4 | Отчеты мастеров СМР | Веб-отчеты | `wl_report_smr` |
| `load_work_order_log_NDT.py` | 5 | Заявки на НК | Excel файлы | `work_order_log_NDT` |

# 9. Технические детали

## 📊 Полный анализ таблицы `сварено_сварщиком`

### Назначение
Агрегированная таблица для статистики по сварщикам и контролю качества.

### Процесс создания

1. **Предварительная обработка**
   - Обнуление клейм при пустой дате сварки

2. **Создание структуры**
   - Основные поля (линия, чертеж)
   - Статистические поля
   - Поля контроля качества

3. **Заполнение данными**
   - Сваренные швы
   - Несваренные швы

4. **Расчет плановых значений**
   - РК, ПВК, МПД, Стило(PMI), ВИК

5. **Заполнение справочных полей**
   - Из LstTp_12460
   - Из основнойНК

### Детальное описание столбцов

#### `годен_стилоскопирование`
- **Источник**: `wl_china.Результаты_Заключения_Стилоскопирование = 'годен'`
- **Смысл**: Количество швов, прошедших стилоскопирование успешно

#### `"Стило(PMI)"`
- **Источник**: Расчет из `основнаяНК` и данных таблицы
- **Смысл**: Плановое остаточное значение (сколько еще нужно проверить)
- **Формула**: Зависит от процента в основнойНК или специального значения '*'

#### `ВИК`
- **Источник**: Копия `заявлен_вик`
- **Смысл**: Количество заявленных на ВИК
