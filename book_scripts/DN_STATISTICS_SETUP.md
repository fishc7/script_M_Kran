# Настройка и использование статистики DN

## Автоматическое создание таблиц

Таблицы статистики DN создаются **автоматически** при первом обращении к странице статистики.

### Как это работает:

1. **При открытии страницы** `http://127.0.0.1:5000/statistics`:
   - Система автоматически проверяет наличие таблиц в базе данных
   - Если таблицы отсутствуют, они создаются автоматически
   - Затем выполняется расчет и сохранение статистики

2. **Функция `ensure_dn_statistics_tables_exist()`**:
   - Проверяет существование всех 5 таблиц:
     - `dn_statistics_daily`
     - `dn_statistics_weekly`
     - `dn_statistics_monthly`
     - `dn_statistics_yearly`
     - `dn_statistics_period`
   - Создает недостающие таблицы с нужными индексами
   - Вызывается автоматически при каждом запросе статистики

### Ручное создание таблиц (опционально)

Если нужно создать таблицы вручную, можно запустить скрипт:

```bash
python scripts/database/create_dn_statistics_table.py
```

Этот скрипт создаст все необходимые таблицы, если их еще нет.

### Автоматическое обновление при запуске

**Статистика DN автоматически обновляется при запуске веб-приложения:**

1. При запуске приложения (через `python app.py`):
   - Автоматически запускается фоновый поток обновления статистики
   - Статистика рассчитывается и сохраняется в БД
   - Процесс не блокирует запуск приложения

2. При первом запросе (если приложение запущено через WSGI):
   - Статистика обновляется при первом HTTP-запросе
   - Используется механизм `@app.before_request` с защитой от повторных вызовов

3. При каждом обращении к странице `/statistics`:
   - Статистика пересчитывается и обновляется в БД
   - Данные всегда актуальны

### Проверка работы

1. Запустите веб-приложение: `python web/app/app.py`
2. В логах должно появиться сообщение: `"Поток обновления статистики DN запущен"`
3. Через несколько секунд: `"Статистика DN успешно обновлена при запуске"`
4. Откройте страницу статистики: `http://127.0.0.1:5000/statistics`
5. Таблицы создадутся автоматически при первом обращении (если их еще нет)
6. Статистика будет рассчитана и сохранена в БД
7. При последующих обращениях данные будут обновляться

### Логирование

Все операции создания таблиц и сохранения статистики логируются:
- Успешное создание: уровень `WARNING` (чтобы было заметно)
- Обновление при запуске: уровень `INFO`
- Ошибки: уровень `ERROR` с полным стеком ошибки
